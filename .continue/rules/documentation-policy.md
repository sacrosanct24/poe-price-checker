DOCUMENTATION POLICY:
1. MINIMIZE documentation files - update existing docs rather than create new ones
2. ONLY README.md should exist in project root directory
3. All other documentation goes in docs/ subdirectories
4. ALWAYS PROMPT before creating any new documentation file
5. Can update existing documentation without asking
6. Consolidate information into fewer, well-organized documents
7. Prefer updating README.md or docs/INDEX.md over creating new files
8. When continuing work: update CONTINUATION.md, don't create session summaries
